# 长青伙伴 · 产品需求文档（PRD）

> 版本：v1.0  
> 文档性质：**功能规格说明**（不含技术选型、不含实现方案）  
> 产品定位：适老化 · 用药提醒 · 健康陪伴  

---

## 1. 文档说明

### 1.1 目的

本文定义「长青伙伴」**要做什么、做到什么算对**，供产品设计、开发、测试统一对齐。

### 1.2 不在本文范围

- 不规定用什么平台（App / 小程序 / 网页等）
- 不规定数据库、服务器、第三方接口品牌
- 不规定开发语言或框架  

以上由技术方案另行决定。

### 1.3 名词解释

| 名词 | 含义 |
| --- | --- |
| **浇水 / 打卡** | 老人在 App 内确认「这一顿/这一次药已经吃了」 |
| **时段** | 早餐前、早餐后、午餐前、午餐后、晚餐前、晚餐后、睡前、随餐 |
| **打卡项** | 一种药 × 一个时段 = 一条待完成项（例如：降压片·早餐后） |
| **长期药** | 无固定结束日，持续出现在计划中 |
| **短期药** | 有疗程天数，到期后自动从计划中移除 |
| **老人模式** | 极简界面，仅保留核心使用路径 |

## 2. 产品背景与目标

### 2.1 背景痛点

- 老人 **经常忘记吃药**
- 市面产品 **操作复杂**，字号小、步骤多
- 老人对「吃药」有 **心理抵触**，不愿被当成「病人」
- 家人难以掌握 **今天到底吃没吃**

### 2.2 产品目标

1. **降低抵触**：用「养树 / 陪伴」替代冷冰冰的「服药任务」
2. **降低门槛**：超大字、大按钮、能点选就不手填
3. **提高依从性**：打卡反馈、连续天数、小树成长
4. **留痕可查**：服药记录、健康单据影像归档

### 2.3 设计原则（必须遵守）

1. **首页是树，不是药单** —— 管理类功能降权到次级页
2. **把治疗感藏起来，把生活感露出来** —— 文案、视觉、动效均遵守
3. **单端合并** —— 当前版本不拆分「老人端 / 子女端」，同一 App 服务两类操作，但 **默认路径以老人为主**
4. **提醒与打卡分离** —— 外部提醒只负责「叫人」；是否已吃 **只在 App 内确认**
5. **识别必须经人确认** —— 拍照识别结果不可自动入库

## 3. 用户与场景

### 3.1 用户角色（逻辑角色，非独立 App）

| 角色 | 典型行为 | 使用频率 |
| --- | --- | --- |
| **老人（主用户）** | 看树、按提示打卡、看夸夸 | 每天多次 |
| **家人（协助者）** | 录入/修改药单、设提醒、管档案、开老人模式 | 偶尔 |
| **（远期）子女远程** | 不在同一设备上管理 | 暂不做 |

### 3.2 核心场景

**场景 A · 日常吃药（老人）**  
到点 → 打开 App → 看「现在」时段 → 吃完点对应药 → 看夸夸 → 小树状态更新  

**场景 B · 一次吃完今天剩余的（老人）**  
打开 App → 点「一键浇水」→ 今日所有未打卡项变为已吃 → 夸夸  

**场景 C · 新处方（家人）**  
拍照或手动添加 → 确认剂量与时段 → 选长期/短期与开始日 → 保存 → 今日/未来计划自动更新  

**场景 D · 复诊（家人 + 老人）**  
打开档案 → 按日期找到化验单/报告照片 → 展示给医生  

**场景 E · 稳定用药的提醒（家人）**  
用药计划稳定 → 同步到手机日历 → 到点提醒老人 **回 App 打卡**

## 5. 功能模块规格

### 5.1 首页 · 养成陪伴（P0）

#### 5.1.1 页面元素

| 元素 | 规则 |
| --- | --- |
| 问候区 | 根据时间段显示早/中/晚好 + 称呼 |
| 引导语 | 无药：温和提示无计划；有未完成：提示还剩几次浇水；全部完成：庆祝文案 |
| 小树展示区 | 显示形态 emoji、形态名、描述、连续天数 |
| 今日用药 | 按 **时段分组** 列表展示 |
| 当前时段 | 系统根据当前时间推断「现在」时段，该组显示 **「现在」** 标记 |
| 每条打卡项 | 药名、剂量、时段、状态标签「还没吃 / 已吃了」、对勾区 |
| 一键浇水按钮 | 未完成时：一键完成今日全部未打卡；全部完成时：置灰/改文案 |

#### 5.1.2 交互规则

| 操作 | 行为 |
| --- | --- |
| 点击「还没吃」项 | 标记为已吃 → 小树短动画 → 弹出夸夸 → 需点「好嘞！」关闭 |
| 点击「已吃了」项 | 弹二次确认 → 确认后撤销为还没吃 |
| 点击一键浇水 | 同批量完成未打卡项，动画 + 夸夸 |
| 打卡成功 | 可选：短震动反馈 |
| 无用药计划 | 显示引导按钮；老人模式下提示家人帮忙，不强制跳转管理页 |

#### 5.1.3 小树成长规则

| 连续天数 | 形态 | 展示名 |
| --- | --- | --- |
| 0～2 | 🌱 | 种子 |
| 3～6 | 🌿 | 小苗 |
| 7～14 | 🪴 | 茁壮幼树 |
| 15～29 | 🌳 | 小树 |
| 30+ | 🌲 | 长青树 |

- 连续天数定义：从今天或昨天起向前数，**有至少一条打卡记录** 的连续自然日
- 断档：某天无任何打卡则连续中断（产品可配置是否宽松，默认严格）

#### 5.1.4 夸夸内容

- 文案池随机，语气温暖、短句、无指责
- **不得** 2～3 秒自动消失；必须用户主动关闭
- 示例：「太棒了，小树又长高了一点！」「吃完药身体棒棒，儿女也放心呀！」

### 5.2 用药管理（P0）

#### 5.2.1 列表页

- 展示所有药物卡片：名称、剂量、时段摘要、长期/短期标签
- 短期药展示：**共 N 天，到 M 月 D 日结束**
- 操作：**修改**、**删除**（按钮足够大，勿用小图标）

#### 5.2.2 添加 / 修改表单

| 字段 | 输入方式 | 校验 |
| --- | --- | --- |
| 药名 | 手填 或 拍照识别带入 | 必填 |
| 剂量 | 步进器 + 单位点选 | 必填 |
| 时段 | 8 个卡片多选 | 至少 1 个 |
| 周期 | 「长期服用 / 短期疗程」二选一 | 必填 |
| 疗程天数 | 短期时：步进器 + 快捷 3/5/7/10/14 天 | 短期必填 |
| 开始日期 | 新增时：「今天 / 明天」 | 必填 |

- 保存后即时影响：**今日用药**、**未来计划**
- 修改已有药：**不改变** 原开始日期（除非产品后续允许改）

#### 5.2.3 拍照识别流程（P0 基础 / P1 增强）

```
拍照 → 识别中 → 大确认卡 → 用户选择 → 进入表单或直接保存
```

**确认卡（必须）：**

| 按钮 | 行为 |
| --- | --- |
| 对的，就这样 | 带入表单或保存 |
| 不对，我自己改 | 打开空表单/预填可改 |
| 让家人帮我改 | 关闭并提示 |

**识别失败 / 不清：** 提示重拍或改手动添加

#### 5.2.4 删除规则

- 二次确认
- 删除后：**历史打卡记录保留**
- 提示：若曾同步日历，需手动到手机日历删除「小树该浇水」类提醒

### 5.3 服药记录（P0）

#### 5.3.1 统计区

- 连续天数
- 总打卡次数
- 小树当前形态（用形态名，非 Lv.数字）

#### 5.3.2 历史记录 · 月历

- 按月浏览
- 当天有打卡：标记 ✓
- 今天：特殊边框
- 图例说明：已服药 / 今天

#### 5.3.3 历史记录 · 今日已服

- 列出今日每条打卡：药名、时段、打卡时间

#### 5.3.4 未来计划

- 默认展示 **未来 7 天**（含今天）
- 按日分组 → 再按时段分组 → 列出药名 + 剂量
- 已打卡的条目在当天显示为已完成样式
- 某药短期到期后，到期日之后不再出现

#### 5.3.5 手机日历提醒（P1，可选能力）

**产品定位：** 辅助叫人回 App，**非** 打卡状态来源。

| 规则 | 说明 |
| --- | --- |
| 合并策略 | 同一 **时段** 合并为 **一条** 每日提醒（避免一种药一条） |
| 提醒标题 | 「🌱 小树该浇水（{时段}）」类，**引导回 App** |
| 提醒说明 | 含称呼、打开 App 浇水、本次涉及药名列表 |
| 默认时刻 | 各时段有默认钟点（如早餐后 8:30），后续可支持自定义 |
| 长期药 | 每日重复，无结束日 |
| 短期药 | 每日重复至疗程最后一天 |
| 同步前 | 展示将添加几条、每天几点、标题预览 |
| 同步后 | 记录同步时间与药单指纹 |
| 药单变更 | 提示「用药有变动，重新同步（慎用）」 |
| 重新同步 | **不** 承诺自动删除旧日历条目；必须提示用户手动清理 |
| 适用建议 | 文案明确：**仅推荐长期稳定用药**；频繁改药不建议依赖 |

**不支持的产品承诺（须写进说明）：**

- 不能通过 App 删除或修改已写入系统的日历事件
- 日历通知被划掉 **不等于** App 内已打卡

## 6. 核心业务规则（汇总）

### 6.1 今日用药如何生成？

对每个 **药物** 的每个 **时段**：

1. 若 **开始日期 ≤ 今天**
2. 且该药 **在今日仍处有效期内**（长期 always；短期 ≤ 结束日）
3. → 生成一条 **今日打卡项**

### 6.2 短期药有效期

- 结束日 = 开始日 + 疗程天数 − 1
- 结束日之后：不出现在今日用药、未来计划

### 6.3 打卡与提醒的关系

```
[外部提醒] → 叫人打开 App
[App 内打卡] → 记录已吃 + 小树成长 + 统计更新
```

二者 **不得** 自动打通。

### 6.4 未来计划生成

- 对从今天起的 N 天（默认 7），逐日调用与「今日用药」相同规则
- 仅展示 **当日至少有一条打卡项** 的日期

## 7. 适老化与文案规范

### 7.1 视觉

- 主色：活力绿
- 字号：显著大于常规 App
- 按钮：大点击区域，关键操作用实心大按钮
- 状态色：「还没吃」暖色提醒；「已吃了」绿色清晰

### 7.2 文案口径

| 避免 | 改用 |
| --- | --- |
| 待服 / 已服 | 还没吃 / 已吃了 |
| 服药提醒 | 小树该浇水 / 打开 App 浇水 |
| 退出登录 | 换个人用 |
| 您已逾期 | 小树有点渴了，记得浇水哦 |

### 7.3 适老化验收清单

**P0（必须）**

- [ ] 添加药以点选为主，仅药名可手填
- [ ] 今日用药按时段分组 + 当前时段高亮
- [ ] 已吃撤销需二次确认
- [ ] 状态字大且口语化
- [ ] 夸夸须用户主动关闭
- [ ] 首页不以添加新药为主视觉

**P1（应有）**

- [ ] 底部导航带图标
- [ ] 老人模式仅 2 Tab
- [ ] 编辑/删除为大按钮
- [ ] 拍照识别确认大卡
- [ ] 打卡震动反馈
- [ ] 树等级用形态名

**P2（可选）**

- [ ] 超时未吃「哄哄」软话推送
- [ ] 重要操作更多无障碍支持

## 8. 非功能需求（产品层）

| 类型 | 要求 |
| --- | --- |
| **易用性** | 老人无协助可完成日常打卡 |
| **可靠性** | 打卡记录不丢；换设备应可恢复（正式版） |
| **安全** | 删除、撤销有确认；识别不可 silent 入库 |
| **隐私** | 健康档案仅用户可见；分享需明示 |
| **容错** | 识别错误可改；日历与 App 状态不一致时有说明 |

## 9. 版本范围

### 9.1 MVP（必须交付）

1. 首页养树 + 按时段打卡 + 夸夸 + 一键浇水  
2. 用药 CRUD + 点选表单 + 长/短期 + 开始日  
3. 记录：统计 + 月历 + 今日已服 + 未来 7 天计划  
4. 老人模式  
5. 设置称呼  
6. 拍照识别 **带确认卡**（识别准确度可迭代，流程必须有）  

### 9.2 v1.1（增强）

1. 健康档案  
2. 手机日历浇水提醒（含慎用提示）  
3. 数据备份 / 换机不丢  

### 9.3 明确不做（当前版本）

- 独立子女端 App / 远程实时监控  
- 化验单数值解析、自动诊断  
- 体重 / 血糖 / 血压曲线  
- 月度健康报告自动生成  
- 多宠物 / 复杂游戏化  
- 在线问诊、购药  

### 9.4 远期预留

- 家人远程查看打卡情况  
- 订阅消息类软提醒  
- 用药时间段自定义钟点  
- 多用户 / 家庭组

## 11. 附录：页面地图

```
[养成/我的树] ← 默认首页
    ├→ （无药时）引导添加
    ├→ 时段分组打卡
    └→ 一键浇水 / 夸夸

[用药]
    ├→ 药物列表
    ├→ 手动添加/修改表单
    └→ 拍照 → 确认卡 → 表单
```
